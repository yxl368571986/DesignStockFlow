# 积分明细弹窗晃动修复 - 测试指南

## 快速测试步骤

### 1. 访问个人中心
```
1. 登录系统
2. 点击顶部导航栏的"个人中心"或头像
3. 进入个人中心页面
```

### 2. 打开积分明细弹窗
```
1. 在个人中心页面，找到"积分余额"区域
2. 点击"查看明细"按钮
3. 弹窗应该打开，显示积分明细
```

### 3. 测试Tab切换（核心测试）
```
依次点击以下Tab，观察弹窗是否晃动：
- 全部
- 充值
- 消费
- 兑换
- 签到
- 任务

✅ 预期结果：弹窗高度保持固定，无任何晃动
❌ 如果失败：弹窗高度变化，产生上下晃动
```

### 4. 测试不同数据状态

#### 测试空数据
```
1. 切换到一个没有数据的Tab（如"兑换"）
2. 应该显示"暂无积分记录"的空状态
3. 空状态应该垂直居中显示
4. 弹窗高度保持不变
```

#### 测试少量数据
```
1. 切换到数据较少的Tab（如"签到"）
2. 表格应该正常显示
3. 表格下方有空白区域
4. 弹窗高度保持不变
```

#### 测试大量数据
```
1. 切换到数据较多的Tab（如"全部"）
2. 表格应该出现滚动条
3. 可以在表格内滚动查看更多数据
4. 弹窗高度保持不变
```

### 5. 测试分页功能
```
1. 如果数据超过20条，应该显示分页器
2. 分页器应该固定在弹窗底部
3. 点击分页器切换页面
4. 弹窗高度保持不变
```

### 6. 移动端测试
```
1. 使用手机或调整浏览器窗口到移动端尺寸（<768px）
2. 重复上述所有测试步骤
3. Tab按钮应该自动换行
4. 弹窗高度应该是450px（比桌面端略小）
5. 所有功能正常，无晃动
```

## 详细测试场景

### 场景1：快速切换Tab
```
操作：快速连续点击不同的Tab
预期：弹窗保持稳定，无任何视觉抖动
```

### 场景2：从有数据切换到无数据
```
操作：从"全部"切换到"兑换"（假设无兑换记录）
预期：
- 表格消失，显示空状态
- 空状态居中显示
- 弹窗高度不变
```

### 场景3：从无数据切换到有数据
```
操作：从"兑换"切换到"全部"
预期：
- 空状态消失，显示表格
- 表格正常显示数据
- 弹窗高度不变
```

### 场景4：滚动表格内容
```
操作：在数据较多的Tab中滚动表格
预期：
- 表格内容可以正常滚动
- 筛选栏和分页器保持固定
- 弹窗整体不滚动
```

## 验收标准

### ✅ 通过标准
- [ ] 切换任意Tab时，弹窗高度保持固定（500px桌面端，450px移动端）
- [ ] 无任何视觉晃动或抖动
- [ ] 空状态垂直居中显示
- [ ] 表格内容超出时出现滚动条
- [ ] 分页器固定在底部
- [ ] 筛选栏固定在顶部
- [ ] 移动端Tab按钮自动换行
- [ ] 所有浏览器表现一致

### ❌ 失败标准
- 切换Tab时弹窗高度变化
- 出现视觉晃动
- 空状态显示位置不正确
- 表格内容无法滚动
- 分页器位置不固定
- 移动端显示异常
