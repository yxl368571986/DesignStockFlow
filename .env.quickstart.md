# 环境变量快速入门

## 🚀 快速开始

### 1. 创建环境配置文件

```bash
# 复制模板创建开发环境配置
cp .env.example .env.development

# 复制模板创建生产环境配置
cp .env.example .env.production
```

### 2. 修改配置

编辑 `.env.development` 和 `.env.production`，根据实际情况修改配置项。

### 3. 启动项目

```bash
# 开发环境（自动加载 .env.development）
npm run dev

# 生产构建（自动加载 .env.production）
npm run build
```

---

## 📝 常用配置项

### 必需配置（必须修改）

```bash
# API地址
VITE_API_BASE_URL=http://localhost:8080/api

# CDN地址
VITE_CDN_BASE_URL=http://localhost:8080
```

### 可选配置（有默认值）

```bash
# 应用标题
VITE_APP_TITLE=星潮设计

# 功能开关
VITE_ENABLE_DEBUG=true
VITE_ENABLE_PWA=false
```

---

## 💡 使用方法

### 方式1：直接访问
```typescript
const apiUrl = import.meta.env.VITE_API_BASE_URL;
```

### 方式2：工具函数（推荐）
```typescript
import { getApiConfig } from '@/utils/env';

const apiConfig = getApiConfig();
console.log(apiConfig.baseURL);
```

## ⚠️ 注意事项

1. **不要提交到Git**
   - `.env.development` 和 `.env.production` 已在 `.gitignore` 中
   - 只提交 `.env.example` 模板

2. **修改后需重启**
   - 修改环境变量后需要重启开发服务器
   - `Ctrl+C` 停止，然后 `npm run dev` 重新启动

3. **安全性**
   - 不要在环境变量中存储密码、私钥等敏感信息
   - 前端环境变量会被打包到代码中，用户可见
