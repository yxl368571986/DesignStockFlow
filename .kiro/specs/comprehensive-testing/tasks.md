## 概述

本任务列表用于系统化地测试和修复星潮设计资源平台的所有功能。测试按照设计文档中定义的顺序执行，每个功能测试同时进行接口规范检查，发现问题立即修复并回归测试。

## ⚠️ 重要测试规范（必读）

### 测试环境要求

**所有测试必须在完整环境下进行：**
1. **数据库服务** - PostgreSQL必须启动并运行
2. **后端服务** - Express后端必须启动 (`cd backend && npm run dev`)
3. **前端服务** - Vue前端必须启动 (`npm run dev`)

**启动顺序：**
```bash
# 1. 确保PostgreSQL服务运行
# 2. 启动后端服务
cd backend && npm run dev
# 3. 启动前端服务（新终端）
npm run dev
```

### MCP工具精准测试规范

**每个测试任务必须使用MCP浏览器工具进行精准测试：**

1. **元素展示测试**
   - 使用 `mcp_chrome_devtools_take_snapshot` 或 `mcp_playwright_browser_snapshot` 获取页面快照
   - 验证每个按钮/元素的存在性和可见性
   - 检查元素文本内容是否正确
   - 验证元素位置是否符合用户习惯（导航在顶部、操作按钮在右侧等）

2. **元素交互测试**
   - 使用 `mcp_chrome_devtools_click` 或 `mcp_playwright_browser_click` 测试点击
   - 使用 `mcp_chrome_devtools_fill` 或 `mcp_playwright_browser_type` 测试输入
   - 使用 `mcp_chrome_devtools_hover` 或 `mcp_playwright_browser_hover` 测试悬停效果

3. **位置验证标准**
   - Logo/品牌标识：左上角
   - 导航菜单：顶部水平排列
   - 搜索框：顶部中央或右侧
   - 用户操作（登录/头像）：右上角
   - 主要操作按钮：页面右侧或底部
   - 返回按钮：左上角或左侧
   - 提交/确认按钮：表单底部右侧
   - 取消按钮：提交按钮左侧

### 浏览器控制台监控规范

**每次页面操作后必须检查控制台：**

1. **监控方法**
   - 使用 `mcp_chrome_devtools_list_console_messages` 获取控制台消息
   - 重点关注 `error` 和 `warning` 类型的消息
   - 使用 `mcp_chrome_devtools_list_network_requests` 检查网络请求错误

2. **错误处理流程**
   ```
   发现错误 → 记录错误详情 → 定位错误源头 → 修复根本原因 → 验证修复 → 回归测试
   ```

3. **常见错误类型及处理**
   - **JavaScript错误**: 检查代码逻辑，修复源文件
   - **网络请求错误(4xx/5xx)**: 检查API接口和后端服务
   - **资源加载错误**: 检查文件路径和资源是否存在
   - **Vue警告**: 检查组件props、响应式数据等
   - **TypeScript类型错误**: 修复类型定义

4. **错误修复原则**
   - 必须从根源解决问题，不能只是隐藏错误
   - 修复后必须验证相同操作不再产生错误
   - 记录修复内容到测试报告

### 按钮跳转逻辑验证规范

**每个可点击元素必须验证跳转逻辑：**

1. **跳转测试流程**
   ```
   点击按钮 → 等待页面加载 → 检查URL → 检查页面状态码 → 检查页面内容
   ```

2. **验证内容**
   - 跳转URL是否正确
   - 页面是否返回200状态码
   - 页面内容是否正确加载
   - 是否出现404/500错误页面

3. **错误页面处理**
   - **404错误**: 检查路由配置、链接地址是否正确
   - **500错误**: 检查后端服务、API接口是否正常
   - **空白页面**: 检查组件加载、数据获取是否成功

4. **跳转类型验证**
   - 站内跳转：验证Vue Router路由正确
   - 外部链接：验证链接有效且在新窗口打开
   - 锚点跳转：验证页面滚动到正确位置
   - 弹窗触发：验证弹窗正确显示

## 错误修复流程

### 控制台错误修复流程

```
1. 发现错误
   ↓
2. 记录错误详情
   - 错误类型（JavaScript/Network/Vue Warning等）
   - 错误消息
   - 错误堆栈
   - 触发操作
   ↓
3. 定位错误源头
   - 根据堆栈信息找到源文件
   - 分析错误原因
   ↓
4. 修复根本原因
   - 修改源代码
   - 不要只是隐藏错误
   ↓
5. 验证修复
   - 重复触发操作
   - 确认错误不再出现
   ↓
6. 回归测试
   - 测试相关功能
   - 确保没有引入新问题
```

### 常见错误类型及修复方法

| 错误类型 | 示例 | 修复方法 |
|---------|------|---------|
| TypeError | `Cannot read property 'x' of undefined` | 添加空值检查，使用可选链 `?.` |
| ReferenceError | `xxx is not defined` | 检查变量声明和导入 |
| Network 404 | `GET /api/xxx 404` | 检查API路由配置 |
| Network 500 | `POST /api/xxx 500` | 检查后端服务和数据库 |
| Vue Warning | `Invalid prop type` | 修复组件props类型定义 |
| CORS Error | `Access-Control-Allow-Origin` | 配置后端CORS |
| Chunk Load Error | `Loading chunk xxx failed` | 检查代码分割配置 |

### 跳转错误修复流程

```
1. 发现跳转问题
   - 404页面
   - 500错误
   - 空白页面
   - 无限重定向
   ↓
2. 检查路由配置
   - 前端路由 (src/router/index.ts)
   - 后端路由 (backend/src/routes/)
   ↓
3. 检查链接地址
   - 确认href/to属性正确
   - 确认动态参数正确
   ↓
4. 检查权限配置
   - 路由守卫配置
   - 用户权限验证
   ↓
5. 修复并验证
   - 修改配置或代码
   - 重新测试跳转
```
