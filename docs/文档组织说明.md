## 🔄 维护指南

### 添加新文档
1. 确定文档类别（测试/部署/安全/性能/任务/项目）
2. 将文档放入 `docs/` 对应的子目录
3. 如果是重要文档，更新 `docs/README.md` 的索引

### 更新现有文档
1. 直接在 `docs/` 相应子目录中修改
2. 更新文档中的日期和版本信息
3. 如果文档路径变更，更新相关引用

### 命名规范
- **英文文档**: 使用 `UPPER_SNAKE_CASE.md`
- **中文文档**: 使用描述性名称，如 `测试修复报告.md`
- **任务文档**: 使用 `TASK_XX_DESCRIPTION.md` 格式
